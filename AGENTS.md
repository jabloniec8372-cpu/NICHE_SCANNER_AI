# NicheScanner AI Development Rules

1. Never delete files unless explicitly approved.
2. Always analyze the project before making changes.
3. Before coding, create a short implementation plan.
4. Wait for approval before modifying files.
5. Make only small, safe changes.
6. Keep backward compatibility whenever possible.
7. After implementation always:
   - summarize modified files,
   - summarize changes,
   - explain any risks,
   - run basic verification when possible.
8. Never change the database schema unless explicitly requested.
9. Keep the project beginner-friendly and well documented.
10. Maintain version history (v0.9, v0.9.1, v1.0, v1.2, etc.).
11. Prefer reusable modules over duplicated code.
12. Do not introduce external dependencies unless approved.
13. Do not add external dependencies without a clear reason and documentation update.
14. Preserve Windows PowerShell compatibility.
15. Keep the terminal CLI menu working.
16. Test the CLI before committing when application behavior changes.
17. Update README.md and ROADMAP.md when features, menu options, version numbers, or project direction change.
18. Keep documentation accurate to the current codebase; do not document features that are not implemented.
19. Keep code and documentation beginner-friendly: clear names, simple explanations, and minimal hidden magic.
20. If something is uncertain, ask instead of guessing.
